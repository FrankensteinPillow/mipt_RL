# HW1

В этой директории находится решение домашнего задания по Reinforcement Learning.

Выбранный вариант для задания 2: `Вариант 2` — определение оптимального порядка развоза пассажиров (`mdp_taxi_v2`).

## Содержимое директории

- `task_1.md` — формальное описание MDP для задачи с одним пассажиром;
- `task_2.py` — реализация `Value Iteration` для варианта 2, инференс и генерация артефактов;
- `task_2_results.json` — результаты запуска `task_2.py` с найденной политикой, историей обучения и маршрутами;
- `task_3.md` — демонстрация работы стратегии и пояснения к результатам;
- `value_iteration_progress.jpeg` — график сходимости `Value Iteration`;
- `mdp_taxi_v1/` и `mdp_taxi_v2/` — выданный код сред и README к ним.

## Как запустить

Основной способ:

```bash
uv run python hw1/task_2.py
```

Если среда запуска ограничивает доступ к стандартному cache `uv`, можно использовать локальный cache:

```bash
UV_CACHE_DIR=/tmp/uv-cache uv run python hw1/task_2.py
```

## Что делает скрипт

После запуска `task_2.py`:

- строится MDP по выданной среде `mdp_taxi_v2`;
- выполняется `Value Iteration`;
- восстанавливается оптимальная политика;
- запускается инференс на нескольких конфигурациях поля;
- перезаписываются файлы `task_2_results.json` и `value_iteration_progress.jpeg`.

В консоль выводится краткая сводка, например:

- число состояний;
- число итераций до сходимости;
- оптимальный порядок заказов для базового поля;
- итоговый reward.

## Как проверить результат

1. Запустить `uv run python hw1/task_2.py`.
2. Убедиться, что скрипт завершился без ошибок.
3. Проверить, что обновились:
   - `hw1/task_2_results.json`
   - `hw1/value_iteration_progress.jpeg`
4. Открыть `hw1/task_3.md` и посмотреть:
   - график обучения;
   - маршруты на нескольких вариантах поля;
   - интерпретацию найденной стратегии.

## Коротко про реализованный алгоритм

В `task_2.py` используется `Value Iteration` с `gamma = 1.0`.

Это корректно для данной постановки, потому что:

- среда детерминированная;
- эпизод конечный;
- после каждого действия число невыполненных заказов уменьшается, то есть циклов в пространстве состояний нет.
